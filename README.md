# Hello Docker — CI/CD Example

Навчальний проєкт для демонстрації контейнеризації застосунку та реалізації
процесів неперервної інтеграції (CI) і неперервної доставки (CD)
з використанням Docker та GitHub Actions.

Проєкт реалізовано у спрощеному вигляді, оскільки основною метою роботи є
пояснення логіки CI/CD, а не створення повноцінної виробничої інфраструктури.

---

## Опис проєкту

Проєкт являє собою простий Flask-застосунок, який запускається у Docker-контейнері
та використовує Redis для зберігання лічильника відвідувань сторінки.
Запуск сервісів здійснюється за допомогою Docker Compose.

---

## Базовий запуск (Docker + Redis)

Redis використовується як сховище для збереження кількості звернень до сторінки.

Запуск застосунку виконується командою:

    docker compose up -d --build

Після запуску застосунок доступний у браузері за адресою:
http://localhost:5000

Зупинка застосунку виконується командою:

    docker compose down

---

## Неперервна інтеграція (CI)

Неперервна інтеграція реалізована за допомогою GitHub Actions.

CI workflow автоматично запускається при push у гілку main та виконує:
- отримання актуальної версії репозиторію;
- збірку Docker-образів;
- запуск контейнерів для перевірки працездатності сервісів;
- коректне завершення роботи контейнерів після перевірки.

Файл конфігурації CI розміщено у стандартній директорії GitHub:

    .github/workflows/ci.yml

---

## Неперервна доставка (CD)

Неперервна доставка реалізована у вигляді автоматизованого deploy-кроку
в GitHub Actions, який виконується після успішного завершення CI.

У рамках навчальної роботи доставка представлена у спрощеному вигляді
та підтверджує готовність застосунку до розгортання без використання
зовнішніх серверів або хмарної інфраструктури.

---

## Пояснення щодо спрощеної реалізації

Проєкт навмисно реалізовано у мінімальній конфігурації, оскільки це:
- спрощує аналіз роботи CI/CD-конвеєра;
- дозволяє зосередитися на ключових принципах автоматизації;
- відповідає вимогам навчального завдання.

---

## Підсумок

- Flask-застосунок контейнеризовано за допомогою Docker;
- Redis використовується як допоміжне сховище даних;
- неперервну інтеграцію реалізовано з використанням GitHub Actions;
- неперервну доставку виконано як автоматизований deploy-крок;
- проєкт відповідає вимогам лабораторних робіт
  «Неперервна інтеграція» та «Неперервна доставка».
